# 店舗データ インポート用テンプレート

`store_template.csv` にお手元のスプレッドシートの内容を整形して貼り付けてください。1行目（ヘッダー）はそのまま、2行目以降のサンプル行は削除して、実データに差し替えてください。

## 列の説明

| 列名 | 必須 | 説明 |
|---|---|---|
| name | ○ | 店名 |
| genre | ○ | ジャンル。下記「既存ジャンル一覧」のいずれかに合わせるのが望ましいですが、無ければ新しいジャンル名でもOK（自動で追加します） |
| region | ○ | ゆかりの島。下記「地域一覧」のいずれか一字一句そのまま入力してください |
| prefecture | ○ | 都道府県（例: 東京都） |
| town | ○ | 市区町村（例: 新宿区歌舞伎町） |
| village | ○ | お店関係者の出身集落・町（例: 笠利町出身） |
| address | ○ | 以降の住所（市区町村より後ろ。例: 歌舞伎町1-2-3） |
| lat / lng | 任意 | 緯度・経度。分からなければ空欄でOK（後で私が住所から調べます） |
| photo_url_1〜3 | 任意 | 写真URL。最大3枚まで（もっと多い場合は列を増やしてもOK） |
| profile | 任意 | 紹介文 |
| store_url | 任意 | 店舗の外部ページURL |
| phone | 任意 | 電話番号 |
| is_ad | 任意 | 広告掲載枠なら `true`、通常は `false` または空欄 |
| reservation_url | 任意 | 予約リンク（is_ad=trueの店舗のみ意味を持つ） |
| provider_note | 任意 | 情報提供者名。入力すると店舗詳細画面に「情報提供: 〇〇さん」と公開表示されます |

## 既存ジャンル一覧（この通りに入力してください。「その他の飲食店」を使う場合は具体的なジャンル名を添えてください）

バー / スナック / 居酒屋（郷土料理あり） / 居酒屋 / 外国料理 / 食堂 / カフェ / その他の飲食店 / 飲食店以外のお店

## 地域一覧（この通りに入力してください）

奄美大島 / 喜界島 / 徳之島 / 沖永良部島 / 与論島 / 加計呂麻島 / 請島 / 与路島 / 未確認（出身島が分からない場合）

## 記入後の流れ

1. `store_template.csv` に実データを記入して保存
2. 私に「記入した」と伝えてもらえれば、内容を読み込んでSupabaseへの一括登録用SQLを作成します
3. その際、既存のサンプルダミーデータ（12件）は削除してから登録します
